# Functional Programming in Education

## Context & Motivation

George Wilson's YOW! 2019 talk on using functional programming for teaching introductory computer science. Focuses on university-level first-year programming courses for complete beginners.

**Core Question:** Which programming language should we teach beginners? Language choice dramatically influences curriculum design and learning outcomes.

## Problems with Traditional Approaches

### Java's Complexity for Beginners
- **Hello World** in Java requires understanding classes, static methods, access modifiers
- Students finish semester without understanding their first program
- **Tyranny of Syntax:** Course structure follows language syntax (Week 3: if statements, Week 4: while loops)
- **Risk:** Students confuse syntax with concepts (thinking they learn "while loops" instead of "iteration")

### Goal of Programming Education
- Dispel notion that computers are "magic" or uncontrollable
- Teach fundamental problem-solving, not just language syntax
- Focus on concepts like iteration, conditionals, data structures

## SICP (Structure and Interpretation of Computer Programs)

### Scheme's Advantages
- **Minimal Syntax:** Only S-expressions + special forms (define, if)
- **Early Achievement:** Factorial function in first lecture using recursion
- **Substitution Model:** Visual evaluation where function calls are replaced by their bodies

### Teaching Benefits
- **Mental Model:** Students can manually trace program execution by substitution
- **Complexity Awareness:** Substitution tree shape shows space/time usage early
- **Broad Coverage:** Symbolic computation, interpreters, logic programming
- **"Tasting Plate" of CS:** Introduces diverse computer science concepts

### Trade-offs
- Examples often too technical (calculus derivatives, digital circuits)
- Dynamic typing provides poor error feedback
- Assignment introduces complex mental models

## Wadler's Haskell Critique (1987)

### Strong Static Typing Benefits
- **Pattern Matching:** Data type definitions directly guide programming
- **Compiler Feedback:** Catches missing cases and incorrect data handling
- **List Example:** `data List a = Nil | Cons a (List a)` - definition becomes programming guide

### Laziness Advantages
- **Evaluation Model:** Normal order evaluation makes substitution model accurate
- **Conceptual Simplicity:** Can define `if` as a function without infinite loops

### Problems with Standard Haskell
- **Error Messages:** "No instance for Num Bool" vs. clear "Can't add booleans and numbers"
- **Complex Concepts:** Type classes confuse beginners

## HTDP (How to Design Programs)

### Staged Learning Approach
- **Progressive Language:** More constructs unlock as students advance
- **Progressive IDE:** Tools adapt to learning stage (simple evaluator â†’ complex debugger)
- **Typed Scheme:** Even advanced version requires proper types (cons second arg must be list)

### Pedagogical Excellence
- **Design Recipes:** Step-by-step function development process
  1. Write function purpose
  2. Create examples
  3. Analyze input data structure
  4. Use data-shaped templates
  5. Convert examples to executable tests
- **Error Message Exercises:** "Pass wrong type to function and analyze error"
- **Documentation Focus:** "Look up the documentation" exercises

### Core Philosophy
Teach programming methodology, not just language syntax. Focus on modularity, specification, data abstraction.

## Author's Vision: Ideal FP Course

### Combine Best of Both Worlds
- **HTDP's Problem-Solving Focus** + **Haskell's Type Safety**
- **Staged Haskell Introduction:** Series of custom Preludes with increasing complexity
- **Custom Error Messages:** Clear, beginner-friendly type errors

### Implementation Approach
- **GHC 8+ Features:** Custom type error messages
- **No Implicit Prelude:** Provide educational Prelude versions
- **Progressive Complexity:** Start with integers-only, add features gradually

### Example Error Improvement
```haskell
3 + False  -- Current: "No instance for Num Bool"
           -- Desired: "Cannot add booleans and numbers"
```

## Personal Perspective

### Learning Experience
- Author's university course was "best educational experience ever"
- Lectures were clear, exercises challenging but achievable
- End-of-course revelation: "I'd rather have taught you Haskell"
- **Key Insight:** Even excellent courses can be improved with better language choice

### Conclusion
Functional programming offers superior teaching tools through:
- **Clear Mental Models** (substitution, pattern matching)
- **Early Complexity Awareness** (space/time visualization)
- **Type Safety** (compiler-guided learning)
- **Progressive Disclosure** (staged language/IDE features)
